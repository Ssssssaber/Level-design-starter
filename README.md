# 2D Top-Down Level Design - Шаблон для студентов

## Обзор

Этот проект - готовый шаблон для создания 2D top-down уровней. Ваша задача:
- Использовать готовые префабы и систему
- Добавить свои визуальные элементы (спрайты, анимации)
- Расставить объекты на сцене для достижения игровой цели

## Быстрый старт

### Что нужно сделать

1. **Откройте проект в Unity 2022.3+**
2. **Создайте новую сцену уровня** в папке `Assets/Scenes/`
3. **Добавьте уровень в данные** - создайте или обновите LevelData
4. **Настройте визуал** - замените спрайты на свои
5. **Расставьте объекты** - разместите врагов, ловушки, головоломки
6. **Протестируйте** - запустите игру и проверьте

### Структура готовых префабов

```
Assets/Prefabs/
├── Player/              # Игрок
├── Enemies/             # Враги (разные типы)
├── Interactables/        # Интерактивные объекты
└── UI/                  # UI элементы
```

## Создание уровня

### Шаг 1: Новая сцена

1. Создайте новую сцену: `File → New Scene → 2D (Core)`
2. Сохраните в `Assets/Scenes/НазваниеУровня.unity`

### Шаг 2: Добавление уровня в игру

1. Создайте или найдите ScriptableObject `LevelData`:
   - В Project: правый клик → Create → Game → Level Data
2. В поле `Level Scenes` добавьте название вашей сцены (без `.unity`)

### Шаг 3: Настройка игровой зоны

Добавьте `FinishGameArea` на объект, который игрок должен достичь:
- Создайте пустой объект или спрайт
- Добавьте компонент `FinishGameArea` (из Scripts/Utility)
- Настройте по желанию

## Настройка визуала

### Замена спрайтов

**Для игрока и врагов:**
1. Найдите нужный объект в сцене или префабе
2. Найдите компонент `SpriteRenderer`
3. Перетащите свой спрайт в поле Sprite

**Для анимаций:**
1. Найдите компонент `Animator`
2. Откройте Animation Window: `Window → Animation → Animation`
3. Создайте новый анимационный клип или замените существующий
4. Добавьте свои спрайты на таймлайн

### Настройка анимаций

Анимации находятся в `Assets/Animations/`. Для изменения:
1. Откройте Animation Window
2. Выберите объект с Animator
3. Редактируйте клипы (Idle, Walk, Attack и т.д.)

## Работа с врагами

### Добавление врага на сцену

1. Перетащите префаб врага из `Assets/Prefabs/Enemies/` на сцену
2. Настройте позицию

### Типы врагов

В проекте есть готовые типы врагов:
- **Melee Enemy** - атака в ближнем бою
- **Ranged Enemy** - атака на расстоянии
- **Patrol Enemy** - патрулирование

### Настройка параметров врага

В компоненте `NPCStateMachine`:
- **Init State** - начальное состояние
- **Vision Zone** - зона обнаружения (настройте радиус)
- **Attack Zone** - зона атаки

### Настройка зон обнаружения

Каждая зона (Vision, Attack, RangedAttack) имеет CircleCollider2D:
- Измените Radius для настройки дальности
- Включите/выключите "Dependent on Vision" для зон

### Враги и головоломки

Враги могут быть частью головоломок (убить всех врагов для открытия двери). Для этого:
1. Добавьте врага в PuzzleManager как Puzzle Part
2. Задайте Required State = Off (враг должен быть мёртв)

## Интерактивные объекты

### Доступные объекты

| Объект | Назначение | Как использовать |
|--------|------------|------------------|
| FloorTorch | Факел | Перетащите префаб, взаимодействуйте для вкл/выкл |
| Door | Дверь | Может открываться/закрываться |
| PuzzleDoor | Дверь головоломки | Связана с PuzzleManager |
| Chest | Сундук | Взаимодействуйте для открытия |
| Key + KeyLockedDoor | Ключ и дверь | Соберите ключ, откройте дверь |
| HealthPotion | Зелье здоровья | Восстанавливает здоровье |
| Note | Записка | Показывает текст при взаимодействии |
| Trap | Ловушка | Наносит урон игроку |

### Настройка объектов

Большинство объектов имеют настраиваемые параметры в Inspector:
- Состояния (On/Off)
- Визуальные спрайты
- Взаимодействие с PuzzleManager

## Головоломки

### Создание головоломки

1. Создайте пустой объект
2. Добавьте компонент `PuzzleManager`
3. Настройте **Puzzle Parts** (что должно измениться):
   - Drag'n'drop объект в Puzzle Object
   - Выберите Required State (On/Off)
4. Настройте **Puzzle Targets** (что изменится):
   - Drag'n'drop объект
   - Выберите Target State

### Примеры головоломок

**Пример 1: Зажги факелы**
- Части: 3 факела, Required State = On (горят)
- Цель: Дверь, Target State = On (открыть)

**Пример 2: Убей врагов**
- Части: Враги, Required State = Off (мёртвы)
- Цель: Проход, Target State = On (открыть)

**Пример 3: Выключи ловушки**
- Части: Ловушки, Required State = Off (неактивны)
- Цель: Безопасный проход

## Tilemaps

### Настройка Tilemap

Проект использует Unity Tilemap для создания уровней:

1. **Создайте Tilemap**:
   - Правый клик в Hierarchy → 2D Object → Tilemap → Rectangular

2. **Добавьте тайлы**:
   - В Project создайте Tile Palette: правый клик → 2D → Tilemaps → Palette
   - Перетащите спрайты в палитру
   - Рисуйте на Tilemap

3. **Настройка коллизии**:
   - Выберите Tilemap
   - В Tilemap Collider 2D настройте коллизию
   - Для навигации врагов - добавьте NavMeshSurface и нажмите Bake

### Навигация (NavMesh)

Для ходьбы врагов по уровню:
1. Добавьте `NavMeshSurface` на сцену
2. Выберите, какие слои учитывать (обычно слой с тайлами)
3. Нажмите **Bake** для построения NavMesh
4. Враги будут автоматически использовать навигацию

### Слои (Layers)

Настройте слои для корректной работы:
- Создайте слой для стен/пола
- Назначьте Tilemap на этот слой
- Настройте Vision Blocking Layers в зонах врагов

## Сборка и тестирование

### Запуск уровня

1. В Unity нажмите **Play**
2. Или запустите из главного меню, выбрав уровень

### Тестирование головоломок

- Проверьте, что все Puzzle Parts достигают нужного состояния
- Убедитесь, что Puzzle Targets активируются правильно

### Отладка

- Используйте Console для просмотра сообщений
- Добавьте Debug.Log для проверки логики

## Частые проблемы

**Враги не ходят:**
- Проверьте, что NavMesh построен (нажмите Bake)
- Убедитесь, что есть путь от врача до игрока

**Взаимодействие не работает:**
- Проверьте, что объект имеет компонент IInteractable
- Убедитесь, что InteractionDetector на игроке

**Головоломка не активируется:**
- Проверьте настройки PuzzleManager
- Убедитесь, что объекты имеют IPuzzleElement
